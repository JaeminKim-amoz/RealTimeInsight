## Goal

Port the existing visual prototype at `public/app/*.jsx` (babel-standalone, 19 files, ~11k LOC) to a TypeScript module architecture per the three RealTimeInsight spec documents (`Master_Prompt`, `UI_Spec`, `Frontend_Component_State_Mock_Spec`), preserving the prototype's visual fidelity (warm graphite + amber, 14-panel Workstation, 4 sheet tabs) **while** wiring it to a real Rust+Tauri backend with a working 10 Mbps PCM data path. The first vertical slice must satisfy spec §22 acceptance criteria in full and a Strict 60-second 10 Mbps performance test.

---

## Constraints

### Architecture
- **Path B**: Port JSX prototype to TypeScript ES modules. The `public/app/*.jsx` files become a visual reference and move to `.design-package/` (already there).
- Stack: Vite + React 18 + TypeScript + Tauri 2 + Rust crates.
- All store contracts follow spec §8: `WorkspaceStore`, `SessionStore`, `SelectionStore`, `StreamStore`, `IntegrationStore`.
- Bridge contract follows spec §11 (`UiBridge` interface + `BridgeEvent` event stream).
- Layout uses **recursive `LayoutNode` tree** (spec §7.1) backing model with a `'workstation-default'` preset that reproduces the prototype's exact 14-panel arrangement using 2-level splits.
- **CDN replacement**: prototype's `unpkg` imports (React, three, maplibre-gl, babel-standalone) replaced with `npm` packages. Spec offline-first principle (§14, §19) honored.

### Data
- 10 Mbps PCM sample data generated by **Rust crate over UDP loopback**: `crates/pcm-generator` → localhost UDP → `crates/pcm-receiver` → bridge `start_managed_receiver_loop` → `StreamStore`.
- Mock data canonical form: **TypeScript source files** at `project/src/mock/*.ts` (deviation from spec §18 JSON — TS chosen for type safety in TDD). Source content is the prototype's `data.jsx` 36 channels + ANOMALY + EVENTS + TRACK_POINTS.
- Browser dev mode (`npm run dev`) keeps `App.tsx`-style `fallbackInvoke` for fast UI iteration. Tauri dev (`npm run tauri:dev`) uses real Rust UDP path.
- Slice 1 bridge-wired channel: **Power Bus Voltage (#1001)** rendered in the first Strip panel; the remaining 13 panels run from ported in-memory mock (the prototype's `bus.jsx` synthesizer).

### Test Stack
- **Vitest + c8** for TS (unit + integration + component via jsdom + RTL). Replaces plain `node *_test.js`.
- **cargo test** + **criterion** for Rust (unit + perf bench).
- TDD pure: **Red → Green → Refactor**, every new TS/Rust file has tests written first, ≥80% coverage on new code via c8.
- Existing `project/tests/*_test.js` files migrate to `tests/integration/` as documentation/migration scripts (not deleted, not extended).

### Cargo Workspace
- Located under `src-tauri/` (Tauri 2 convention).
- Members: `pcm-generator`, `pcm-receiver`, `tauri-shell`.

### Subscriptions
- Subscription lifecycle owned by `StreamStore` (spec §8.4). Components subscribe via `useStreamSubscription(channelIds, schema)` hook, never directly call bridge.

### Drag-Drop
- 5-zone overlay from prototype kept (`center / top / bottom / left / right`).
- Decision table from spec §12.3 applied per panel kind.

### Visual Fidelity
- Warm graphite + amber theme retained verbatim (`public/app/styles.css` ported to `project/src/styles/`).
- IBM Plex Sans + JetBrains Mono fonts (loaded from `npm` `@fontsource` packages, not Google CDN — offline-first).
- All 14 panels of `INITIAL_PANELS` from prototype recreated in `'workstation-default'` preset.
- 4 sheet tabs (`Workstation / Space / EW / GPS LOS`) all present and switchable in slice 1; only Workstation has full panels — the other 3 render their existing layouts (Globe, EW Theater, GPS LOS) but read from in-memory mock only.

- The `hexmap.jsx` and `rf.jsx` files from the prototype (not loaded in current `index.html` either; slice 2+).
